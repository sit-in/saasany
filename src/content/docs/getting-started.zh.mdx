## 前置要求

开始之前，请确保你的开发环境满足以下条件：

- **Node.js** 18.17 或更高版本
- **npm** 或 **pnpm** 包管理器
- **Git** 用于克隆仓库
- 一个 [Neon](https://neon.tech) 数据库（免费额度足够开发）

## 克隆项目

```bash
git clone https://github.com/sit-in/saasany.git my-saas
cd my-saas
```

## 安装依赖

```bash
npm install
```

## 配置环境变量

复制环境变量模板文件，然后填入你的密钥：

```bash
cp .env.example .env.local
```

最小可运行配置只需要三个变量：

```bash
# 数据库连接（从 Neon 控制台复制）
DATABASE_URL=postgresql://user:password@host/database?sslmode=require

# 认证密钥（至少 32 个字符的随机字符串）
BETTER_AUTH_SECRET=your-secret-key-at-least-32-characters
BETTER_AUTH_URL=http://localhost:3000

# 应用地址
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

> 其他环境变量（Stripe、OpenAI、Resend 等）可以后续按需配置，不影响项目启动。

## 初始化数据库

将 Drizzle schema 推送到 Neon 数据库：

```bash
npx drizzle-kit push
```

这条命令会根据 `src/lib/db/schema.ts` 中的定义自动创建所有表。

## 启动开发服务器

```bash
npm run dev
```

打开 [http://localhost:3000](http://localhost:3000)，你应该能看到落地页。

## 验证一切正常

快速检查清单：

1. 首页正常显示，中英文切换无报错
2. 点击「注册」可以创建账号
3. 登录后能进入 Dashboard
4. 终端没有数据库连接错误

## 首次部署

最快的部署方式是 Vercel：

```bash
npm i -g vercel
vercel
```

部署时在 Vercel 控制台添加所有 `.env.local` 中的环境变量。详细部署说明请参考[部署文档](/docs/deployment)。

## 项目结构概览

```
src/
├── app/              # Next.js App Router 页面
│   ├── [locale]/     # 国际化路由
│   │   ├── (auth)/   # 认证页面（登录/注册）
│   │   ├── (dashboard)/ # 需登录的页面
│   │   └── (marketing)/ # 公开页面
│   └── api/          # API 路由
├── components/       # React 组件
├── config/           # 站点配置、主题、定价
├── i18n/             # 国际化配置和翻译文件
└── lib/              # 工具函数、数据库、认证
```
